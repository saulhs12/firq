# Support

## Getting help

- Open a GitHub issue for bugs or feature requests.
- Use discussions (if enabled) for design questions and usage guidance.

## What to include in support requests

- Rust version (`rustc --version`)
- Platform details (OS and architecture)
- Minimal reproducible example
- Relevant logs or benchmark output

## Scope

Firq is an in-process scheduler workspace.
Support focuses on the published crates and examples in this repository.
